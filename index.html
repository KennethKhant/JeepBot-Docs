VESC, Encoder, F710 Controller & Raspberry Pi Setup

Reference guide for setting up the JeepBot steering encoder, configuring VESC Tool, installing PyVESC, preparing the Raspberry Pi, and running the F710 dual VESC control script.

01

Setup Encoder

In VESC Tool, set the motor type to DC motor.

Under Motor Settings → General → Motor Type → DC
Motor Type: DC, Brushed Motor
Current
Adjust motor and battery current limits if needed. If the battery cannot handle the load or the VESC red light blinks, adjust the current settings.
02

VESC App Configuration

Enable the encoder and set the VESC app to UART so the Raspberry Pi Python script can send commands through serial.

Additional Info → General → Position Sensor
Set Position Sensor to Encoder.
App Settings → General
Set App to Use to UART.
03

PID Position Setup & Real-Time Test

Use real-time data in VESC Tool to check PID position control. Set the PID position offset under Motor Settings, then write both app config and motor config.

Data Analysis → Real-time Data → PID Pos
Move the motor with left/right arrow keys and check encoder feedback.
Motor Settings → PID Control
Tune PID values until the steering motor responds smoothly.
Important
Click Write App Config and Write Motor Config after changes.

05

Test on Laptop

Clone the PyVESC repository and install it in editable mode before running the JeepBot script.

git clone https://github.com/LiamBindle/PyVESC.git
cd PyVESC
pip install -e .

cd ~/JeepBot_Vesc/dual_vesc_drive_package
pip install -r requirements-steering.txt
06

Run on Raspberry Pi

After the project folder is copied to the Raspberry Pi, activate the correct Python environment, go into the JeepBot controller folder, install the required packages, and run the controller script.

Step 1: SSH & Activate the Python environment

This makes sure the script uses the correct installed packages.

ssh jeepbot@192.168.139.223
Switch env jeepbot 
source ~/jeepbot/bin/activate
source ~/env/bin/activate
Step 2: Go to the JeepBot controller folder

This folder contains the main controller file, pygame joystick file, and requirements file.

cd ~/myjeepbot/mycar
Step 3: Install required packages

Install the Python packages needed for serial communication and controller input.

pip install pyserial pygame
pip install -r requirements-steering.txt
Step 4: Test the controller only

This checks if the Logitech F710 is detected before sending commands to the VESCs.

python3 myconfig.py --dry-run --no-deadman
Step 5: List available VESC ports

The VESCs should appear as /dev/ttyACM0 and /dev/ttyACM1.

python3 myconfig.py --list-ports
Step 6: Run JeepBot

Run the controller script using the saved configuration.

python3 myconfig.py
07

Troubleshooting

Use these commands if the controller, VESC ports, or motor movement is not working correctly.

Check VESC ports

If the VESC is connected correctly, it should show up as /dev/ttyACM0 or /dev/ttyACM1.

python3 myconfig.py --list-ports
ls /dev/ttyACM* /dev/ttyUSB*
Test controller input only

This shows joystick axis values without moving the motors.

python3 myconfig.py --dry-run --no-deadman
Test known working drive settings

Use this if the motor does not spin with the default settings.

python3 myconfig.py --no-deadman --drive-axis 3 --max-duty 0.30 --drive-ramp-duty-per-s 2.0
Fix inverted driving

If forward and backward are reversed, add the invert-drive option. Then swap that in myconfig.py

python3 myconfig.py --invert-drive
Swap VESC ports

If steering and drive are connected to the wrong VESC, swap the ports. Then swap that in myconfig.py

python3 myconfig.py --steering-port /dev/ttyACM1 --drive-port /dev/ttyACM0
